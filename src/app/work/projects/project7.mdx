## Overview

This project explores the use of blockchain technology to give patients control over their medical history, ensuring privacy and seamless data transfer between different healthcare providers.

## Key Features

- **Patient Sovereignty:** Patients grant and revoke access to their records.
- **Immutable Audit Logs:** Every access request is recorded on the blockchain.
- **Interoperability:** Standardized FHIR data formats for compatibility.

## Technologies Used

- **Blockchain:** Hyperledger Fabric
- **Backend:** Go, Node.js
- **Frontend:** Next.js

## Outcome

Aims to eliminate fragmented medical records and improve diagnosis speed by making patient history readily available securely.
